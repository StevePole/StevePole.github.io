Request Format

Bad: 
curl -X POST \
-F 'a=b' \
-F 'c[d][]=1' \
https://api.example.com/example

Older web servers expect individual parameters to be supplied in POST/PUT data, with string encode JSON used for individual parameters, as seen below.

Better: 
curl -X POST \
-F 'a=b' \
-F 'c={"d":[1]}' \
https://api.example.com/example

Ideally you should allow users to simply post string encoded JSON as follows.

Best: 
curl -X POST \
-D '{
    "a": "b"
    }
    "c": {
        "d": [1]
    }
}' \
https://api.example.com/example

You may also choose to allow users to supply/request XML, it's not highly recommended, XML contains a number ways to encode the same relationships e.g. attributes vs elements, various array definitions/structures, so the transition back to JSON can become indeterminate.
